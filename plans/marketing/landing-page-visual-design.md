# Flux Landing Page: Visual Design

## Status

Accepted (v0 visual direction)

---

## Objective

Define the visual language of the Flux landing page.

This document ensures:

- consistency across implementations
- alignment with product tone
- minimal, intentional design decisions

---

## Core Principle

```txt
Calm > flashy
Simple > impressive
Whitespace > components
```

---

## Design Mood

Flux should feel:

```txt
technical
calm
editorial
intentional
```

---

## Avoid

```txt
loud gradients
heavy animation
busy dashboards in hero
SaaS-style feature overload
```

---

## Font Pairing

### Final Selection

```txt
Headlines: Geist Sans
Body/UI:   IBM Plex Sans
Code:      IBM Plex Mono
```

---

## Why

- Geist Sans → keeps headlines clean and modern
- Sans → keeps UI readable and neutral
- Mono → reinforces developer context

---

## Typography Rules

### Hero

```txt
sans
large scale
medium weight (not heavy bold)
tight tracking
```

Tone:

```txt
editorial, not marketing
```

---

### Body Text

```txt
sans
muted color
relaxed line height
```

Should feel:

```txt
easy to read
not dense
not loud
```

---

### Bullets

```txt
sans
simple
even spacing
no icons
```

---

### CLI Block

```txt
mono
small text
subtle background
minimal emphasis
```

Tone:

```txt
hint of real usage
not a demo
```

---

## Layout Structure

```txt
Hero
↓
Bullets
↓
CLI snippet
↓
Lifecycle
↓
CTA
```

---

## Container

```txt
centered
narrow width (~640–720px)
generous horizontal padding
```

---

## Spacing

Use vertical spacing as primary design tool:

```txt
large gaps between sections
clear separation of ideas
no cramped layouts
```

---

## Visual Rhythm

The page should feel like:

```txt
Headline

(space)

Subtext

(space)

CTA

(space space)

Bullets

(space space)

CLI

(space space)

Lifecycle

(space space)

CTA
```

---

## CLI Block Design

Should be:

```txt
small
quiet
centered
```

Avoid:

```txt
large dark terminal blocks
complex styling
multiple commands
```

---

## CTA Styling

### Top CTA

```txt
simple
neutral size
clearly visible
```

---

### Bottom CTA

```txt
slightly more emphasis
same style
not oversized
```

---

## Color Philosophy

Keep palette:

```txt
neutral
minimal
non-distracting
```

Design should not compete with content.

---

## Content Density Rule

```txt
If it feels like a lot to read, it is.
```

---

## Final Visual Goal

The page should feel:

```txt
effortless
calm
trustworthy
```

Not:

```txt
impressive
complex
feature-heavy
```

---

## Acceptance Criteria

The design is successful if:

- it feels simple at a glance
- nothing feels “designed for attention”
- content is readable instantly
- user is not overwhelmed

---

## Final Reminder

```txt
The product is complex.
The interface should not be.
```
